# Contributing

Thank you for considering a contribution to LympClear.

## Before opening a pull request

- make sure no clinical or identifying data are included;
- avoid committing model weights unless the repository policy explicitly allows it;
- prefer adding new public-facing utilities under `scripts/`;
- document any new dependency in `requirements.txt`;
- update `README.md` or `docs/` when behavior changes.

## Code style

- keep paths configurable rather than machine-specific;
- prefer command-line arguments over hard-coded local folders;
- add concise docstrings for public helper scripts;
- preserve traceability when refactoring research scripts.

## Issues and feature requests

When reporting an issue, include:

- operating system;
- Python version;
- full command used;
- complete error message;
- whether the problem is in public-facing scripts or original research scripts.

## Security and privacy

Do not submit patient data, identifiable screenshots, protected metadata, or confidential institutional material in issues or pull requests.
